# 자동화 흐름 (automation-flow.md, §8·§20·§21)

## 목표
매일 **최대** 5개 콘텐츠 생성. 기준 미달이면 적게. 저품질로 채우지 않는다.

## 권장 구성 (하루)
1 공식 최신 소식 · 2 공연/방송 일정 · 3 가수 프로필/연대기 ·
4 신곡/대표곡/앨범 · 5 주제별 종합 가이드.

## 파이프라인 단계 (lib/automation.ts)
1 공식 출처 목록 수집 → 2 새 정보 탐지 → 3 중복 확인 → 4 기존 콘텐츠 비교 →
5 출처 신뢰도 평가 → 6 사실 추출 → 7 유형 결정 → 8 초안 생성 →
9 제목·본문 중복 검사 → 10 명예훼손·사생활·저작권 위험 검사 →
11 출처 링크 유효성 → 12 이미지 생성/보류 → 13 SEO 메타 생성 →
14 내부 링크 연결 → 15 품질 점수 → 16 발행 or 검수 대기 →
17 RSS·사이트맵 갱신 → 18 오류 로그.

## 자동 발행 조건 (모두 충족)
신뢰 출처 존재 · 날짜/인물/행사명 명확 · 중복 아님 · 위험 키워드 없음 ·
링크 정상 · 제목/본문 일치 · 이미지 권리 문제 없음 · 품질 ≥85 · 구조화 데이터 오류 없음.

## review 강제 (자동 발행 금지, §8)
논란·사건사고·건강·사망·법적분쟁·가족관계·연애결혼·재산출연료·정치발언·
단일출처 민감·익명관계자·SNS 삭제글·팬커뮤니티발.

## 마스터 스위치
`AUTO_PUBLISH_ENABLED=false` 기본. 대표 콘텐츠·파이프라인 검증 전까지 자동 발행 비활성(§24).
false면 모든 후보는 `draft`/`review`로만 저장.

## 실행 방식
- 엔드포인트 `POST /api/cron` (헤더 `Authorization: Bearer <CRON_SECRET>`).
- 중복 실행 방지 락(같은 날 1회). 실패 작업 재시도 횟수 제한.
- 스케줄러: Vercel Cron 또는 OS 스케줄러/`node-cron`.
