# 콘텐츠 데이터 모델 (content-model.md)

전체 스키마는 `prisma/schema.prisma`. 여기서는 의미와 규칙을 정의한다.

## 공통 규칙
- 확인되지 않은 값은 **null**로 저장하고 화면에서 **숨긴다**(추정 금지, §4·§10).
- 배열형 필드는 SQLite 제약상 **JSON 문자열**로 저장(`@/lib/json`에서 파싱).
- enum 성격 필드는 String + `@/lib/enums` 허용값 상수.
- 모든 SAMPLE/DEMO 데이터는 `isSample=true`로 표시(§24).

## 엔터티

### Artist (가수, §4)
`slug, stageName, realName?, profileSummary, birthDate?, birthPlace?, debutDate?, agency?, fanClubName?, officialWebsite?, officialSocialLinks[], awards[], timeline[], music[], events[], programs[], articles[], sources[], lastFactCheckedAt?, status`
- 인기 순위 필드 없음(임의 순위 생성 금지).

### Article (기사/소식, §6·§10)
`type, slug, title, seoTitle?, description, excerpt?, body, artists[], programs[], events[], music[], categories[], tags[], searchIntent?, primaryKeyword?, relatedKeywords[], publishedAt?, updatedAt, lastFactCheckedAt?, nextReviewAt?, timeSensitive, author?, reviewer?, sources[], heroImage?, heroImageAlt?, isAiImage, status, canonical?, index, riskLevel, qualityScore?, qualityBreakdown?, autoGenerated`
- `type`: news | release | broadcast | award | announcement | roundup | guide | timeline

### Event (공연/행사, §5)
`slug, eventName, eventType, startDateTime, endDateTime?, venue?, address?, region?, ticketVendor?, ticketUrl?, ticketOpenDate?, priceInformation?, ageRestriction?, organizer?, officialSourceUrl?, transportInfo?, eventStatus, cancelledAt?, rescheduledFrom?, sourceCheckedAt?, lastUpdatedAt`
- `eventStatus`: scheduled | ticket_open | sold_out | cancelled | postponed | completed | verification_needed
- 취소·연기 시 **삭제 금지** → 상태 변경 + 상단 안내.

### Music (음반/곡)
`slug, title, type(single|album|ost), releaseDate?, label?, description?, trackList[], artists[], sources[]`

### Program (방송 프로그램)
`slug, name, broadcaster?, airInfo?, artists[], articles[]`

### Source (출처, §9)
`sourceTitle, sourcePublisher?, sourceUrl, publishedAt?, accessedAt, sourceGrade(A|B|C), factType?, quotedText?, verificationStatus, linkOk` + 소속(article/artist/event/music)
- **기사 본문 전체를 저장·복제하지 않는다.** 사실과 출처 메타만 저장.

### 운영 로그
- `RevisionLog` 변경 내역(§18), `CorrectionRequest` 정정 요청(§18),
  `AutomationLog` 파이프라인 로그(§21), `AdminActivityLog` 관리자 활동(§22).

## 상태 워크플로 (§16)
`draft → researching → verification_needed → review → scheduled → published`
(그 외 `rejected`, `archived`). 민감 콘텐츠는 관리자 승인 없이는 `published` 불가(§8).
