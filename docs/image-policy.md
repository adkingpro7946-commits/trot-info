# 이미지 정책 (image-policy.md, §11)

## 원칙
- 실제 가수 사진을 무단 수집·재게시하지 않는다.
- 허가 미확인 언론사 사진/방송 캡처/공연 포스터/SNS 사진을 저장·재사용하지 않는다.
- **실제 가수의 얼굴을 사실적 보도사진 형태로 임의 생성하지 않는다.**

## 이미지 생성 전 판단 (§11)
1. 실제 얼굴이 반드시 필요한가?
2. 공식/사용 허가된 사진이 있는가?
3. 사용 권리를 확인할 수 있는가?
4. 일반 무대/마이크/조명 이미지로 대체 가능한가? → 기본은 **대체**.

## 기본 스타일 (OpenAI Images API)
트로트 공연 무대 / 마이크·조명 / 관객·공연장 / 편집 일러스트 /
식별 어려운 실루엣 / 정보 카드형 그래픽 / 세련된 K-대중음악 매거진 스타일.
- 이미지 내부에 긴 한글 제목을 직접 생성하지 않는다.
- 제목·날짜·가수명은 **HTML/CSS/SVG 텍스트 레이어**로 별도 표기.
- 필요 시 "이해를 돕기 위해 제작된 이미지입니다." 캡션(`isAiImage=true`).

## 규격
- 대표 이미지 1200×630, WebP/AVIF, width/height 명시, 반응형 srcset, 고유 alt.

## 키·실패 처리
- `OPENAI_API_KEY`는 env로만 관리(코드/깃 금지).
- 생성 실패 시 발행 중단하지 않고 **기본 카테고리 이미지** 사용 또는 검수 대기 전환.
- 프롬프트는 인물 지정 대신 분위기/구도만 기술(`lib/image.ts` buildPrompt).
