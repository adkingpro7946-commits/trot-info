# 법적·사실 확인 위험 체크리스트 (legal-risk-checklist.md)

## 저작권 위험
- [ ] 언론사 기사 문장 복제/치환 재작성 아님 (§0-1,2)
- [ ] 제목·본문·사진·포스터 무단 복제 아님 (§0-3)
- [ ] 이미지가 허가 미확인 사진(언론/방송캡처/포스터/SNS)이 아님 (§11)
- [ ] AI 이미지가 실제 보도사진으로 오인되지 않음, 필요 시 캡션 표기 (§11)
- [ ] 직접 인용은 원문 확인 + 짧게 + 출처 표기 (§0-11)
- [ ] 출처 저장은 사실·메타만, 기사 전문 저장 안 함 (§9)

## 명예훼손·사생활 위험
- [ ] 확인 불가 나이/가족/학력/재산/건강/연애/결혼/갈등/논란/출연료/수입 생성 안 함 (§0-6)
- [ ] 루머·사생활 침해·악성 의혹·미확인 사건 콘텐츠화 안 함 (§0-7)
- [ ] 범죄·소송·논란은 공식 판결/수사발표/당사자·소속사 입장 없이 자동 발행 안 함 (§0-8)
- [ ] 팬카페·커뮤니티·댓글을 사실 근거로 쓰지 않음 (§0-9)
- [ ] 없는 인터뷰 발언을 만들지 않음 (§0-10)
- [ ] 가수 간 임의 인기 순위 생성 안 함 (§4)

## 사실 정확성 위험
- [ ] 중요한 사실 2개 이상 교차 확인 (§0-5)
- [ ] 날짜/인물/행사명 일치 (§20)
- [ ] 확인 불가 값은 null + 화면 숨김 (§4·§10)
- [ ] 변경 가능 일정에 확인 기준일 표기 (§7)
- [ ] 공연 취소/연기 시 삭제 대신 상태 변경 + 상단 안내 (§5)

## 상거래 오인 위험
- [ ] 티켓 직접 판매처럼 표현 안 함 (§5)
- [ ] 가격/잔여좌석은 실시간 아닐 수 있음 안내 (§5)
- [ ] 검색 노출 보장 문구 없음 (§12)
- [ ] '실시간 인기/가장 많이 본'을 실제 데이터 없이 표기 안 함 (§15)

## 자동 발행 차단 조건 (하나라도 해당 시 review, §20)
출처 없음 / 날짜 불일치 / 가수명 불일치 / 미확인 인용 / 기존 기사 높은 유사도 /
민감 주제 / 이미지 권리 불명확 / 깨진 공식 링크 / 구조화 데이터 오류 / 품질 <85.

## 보안 (§22)
- [ ] API 키 env 관리, 깃 제외
- [ ] 관리자 로그인 보호 + 비밀번호 해시 + CSRF/XSS 방어 + 입력검증 + rate limit
- [ ] cron 중복 실행 방지 + 재시도 제한 + 활동 로그 + 백업
