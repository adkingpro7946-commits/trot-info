# 출처 정책 (source-policy.md, §0·§9)

## 원칙
1. 기존 언론사 기사를 복사하거나 문장만 바꾸어 재작성하지 않는다.
2. 한 기사만 근거로 새 기사를 만들지 않는다. 중요한 사실은 **2개 이상 교차 확인**.
3. 제목·본문·사진·포스터를 무단 복제하지 않는다.
4. 확인 불가한 나이/가족/학력/재산/건강/연애/결혼/갈등/논란/출연료/수입을 추측·생성하지 않는다.
5. 팬카페·커뮤니티·SNS 댓글은 사실 확인 출처로 사용하지 않는다.
6. 실제 인터뷰가 없으면 발언을 만들지 않는다. 직접 인용은 원문 확인 시에만 짧게 + 출처 표시.

## 출처 등급
### A등급 (1차·공식)
가수/소속사 공식 홈페이지, 공식 SNS, 방송사 공식 편성표, 공연 주최사,
공식 티켓 판매처, 음원 플랫폼 공식 앨범 정보, 공공기관, 공식 보도자료.

### B등급 (취재 주체·작성자 명확)
언론사, 방송사 연예뉴스, 전문 음악 매체.

### C등급 (사실 확인 출처로 사용 금지)
출처 불명 블로그, 커뮤니티, 팬 계정, 영상 재편집 채널, 출처 없는 SNS.

## 교차 확인 규칙
- **민감/중요 사실**: A등급 1개 이상 + 서로 독립된 출처 2개 이상.
- 출처가 서로 다르면 차이를 본문에 설명한다.
- C등급 단독 근거 → 자동 발행 불가(review로만).

## 저장 항목 (Source 모델)
`sourceTitle, sourcePublisher, sourceUrl, publishedAt, accessedAt, sourceGrade, factType, quotedText, verificationStatus, linkOk`
- 기사 전문 저장 금지. 필요한 사실 + 출처 메타만.

## 자동 판정 (lib/sources.ts)
- 등급 A/B만 factType별 검증 카운트에 포함.
- verificationStatus: `verified`(A 1개+), `cross_checked`(독립 2개+), `unverified`(그 외).
- 링크 유효성(HTTP 200)은 발행 전·정기 점검에서 확인, 실패 시 `linkOk=false`.
