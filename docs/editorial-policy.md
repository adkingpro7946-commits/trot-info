# 편집 정책 (editorial-policy.md, §6·§7·§13)

## 콘텐츠 유형 (§6)
A 신곡·앨범 / B 방송 출연 확정 / C 콘서트·행사 / D 수상·공식 기록 /
E 소속사 공식 발표 / F 프로그램 출연 결과 / G 가수 활동 정리 / H 주간 일정 종합.

## 기사 기본 구조 (§6)
1) 제목 2) 2~3문장 핵심 요약 3) 무엇이 공식 발표됐는가 4) 날짜·시간·장소 핵심 정보
5) 기존 활동과의 관련 6) 팬 확인 사항 7) 관련 가수 프로필 8) 관련 공연·방송·음반
9) 공식 출처 10) 마지막 확인 시각 11) 정정 요청 안내.

## 본문 기준 (§7)
- 문장 늘리기·같은 정보 반복 금지, 과장 수식어 최소화.
- 사실/해설/추정을 구분하고, 사실에는 출처 연결.
- 날짜는 '오늘/내일' 대신 **절대 날짜** 병기. 변경 가능 일정엔 확인 기준일 표기.
- 출처가 다르면 차이를 설명. 정보 부족 시 억지로 늘리지 않는다.
- 권장 분량 고정 없음. 필요한 만큼만.

## 제목 규칙 (§6·§13)
- 페이지당 중심 검색 의도 1개. 가수명/사건/날짜/장소/프로그램명을 자연스럽게 포함.
- 클릭을 유도하되 본문이 약속한 정보를 실제로 제공.
- **금지 제목**: "충격적인 근황 / 믿을 수 없는 소식 / 팬들 모두 오열 / 결국 터졌다 /
  재산이 공개됐다 / 결혼설의 진실 / 누가 더 잘나가나" 및 확인 안 된 관계·갈등 암시.
- '최신/총정리/완벽 가이드'를 기계적으로 붙이지 않는다. 연도는 일정·방송·공연 글에만.

## 민감 주제 (review 강제, 자동 발행 금지, §8)
논란·사건사고·건강·사망·법적분쟁·가족관계·연애결혼·재산출연료·정치발언·
단일출처 민감기사·익명관계자·SNS 삭제글·팬커뮤니티발 주장.
→ `riskLevel=high`, 상태는 최대 `review`. 관리자 승인 필수.

## 발행량 원칙 (§8·§13)
매일 5개는 **목표**지 의무가 아니다. 기준 통과분만 발행. 부족하면 그 날은 적게.
같은 가수 유사 기사를 하루 여러 개 발행하지 않는다.
